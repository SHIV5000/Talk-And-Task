Key Components
App.jsx Top-level

Purpose: Authentication entry point. Shows Google login screen when no user; after login renders ChatApp inside ErrorBoundary.

State: user, authChecked, authError

Main logic: handleGoogleLogin – initializes persistence, signs in with popup, writes/updates user doc in Firestore.

ChatApp.jsx Core orchestrator

Purpose: The brain of the app. Holds all state, business logic, and renders the entire chat/admin UI. It still contains JSX for LeftSidebar, RightSidebar, chat view, and all modals inline.

Props: user, onLogout

Key states: messages, groups, dbUsers, activeGroup, viewMode, activeModal, searchQuery, chatFilter, selectedMessage, replyingTo, etc. (over 30 states)

Main sub‑functions: handleSendMessage, convertToTask, handleReaction, handleProfileSubmit, handleGroupSubmit, handleCompleteTask, etc.

Renders: AdminPanel (when viewMode="admin"), LeftSidebar inline, RightSidebar inline, chat message list, input area, typing indicator, and all modal components via activeModal.

AdminPanel.jsx

Purpose: Admin dashboard: immutable audit log table, user management, department management.

Props: setViewMode, setActiveModal, dbUsers, groups, filteredAuditLogs, filter state & setters, handleDownloadAudit, handleToggleApprove, setAdminForm, setGroupForm, setEditingGroup, handleAdminArchiveGroup, handleAdminRecoverGroup

MemoizedAvatar.jsx

Purpose: Cached avatar component using localStorage. Shows user/group image or initials.

Props: uid, url, name, sizeClass, isGroup, extraClasses

UploadOverlay.jsx

Purpose: Displays a full‑screen spinner while a file is uploading.

Props: uploadProgress, fileName (unused)

ErrorBoundary.jsx

Purpose: Catches React render errors and shows a reload button.

Core Functions (in ChatApp.jsx)
handleSendMessage

Sends a text message, handles @mentions (creates private mentions & DM forwards), attaches reply data, clears typing indicator, logs audit.

convertToTask

Converts selectedMessage to a task by setting isTask=true and taskData (assignees, deadline, trail).

handleCompleteTask / handleArchiveTask / handleDelegateTask

Update task status, archive, or reassign. Logs audit trail.

handleReaction

Adds/removes an emoji reaction, notifies the original sender.

logImmutableAction

Writes a record to the audit_logs collection with action type, user, and target.

handleProfileSubmit

Updates user display name, font, tool preferences, and optionally uploads a new profile picture to Storage.

handleGroupSubmit / handleUpdateGroupMembers

Create or edit a department group, save members/admins, manage avatar.

getUnreadInfoForUser

Calculates unread message count and pending task count for a specific DM user.

formatMessageText

Converts *bold*, _italic_, ~strikethrough~ markers to HTML (used in renderMessageNode).

playAlertSound / playTaskSound

Plays notification sounds based on user's sound profile preference.

Feature Flow Maps
📬 Send Message

User trigger: Type text & press Enter / click send icon

Component: Chat input area (inline in ChatApp.jsx)

File: ChatApp.jsx → handleSendMessage

Fields involved: inputText, activeGroup, replyingTo, mentions

Flow:

  1. Validate inputText and activeGroup.
  2. Extract @mentions from text, determine if private.
  3. Build message object with text, senderUid, groupId, reply data, etc.
  4. Write to messages collection via addDoc.
  5. Clear typing indicator.
  6. If group is DM, send notification to other user.
  7. If private mentions, forward the message as a DM to each mentioned user.
  8. Log audit action MESSAGE_CREATE.

📎 File Upload (paste or attach)

User trigger: Paste an image or click the "+" button.

Component: Inline input area, hidden file input.

File: ChatApp.jsx → uploadFileDirectly

Fields: file object, activeGroup

Flow: Upload to Firebase Storage (chat_uploads/...), get download URL, write message with fileUrl/fileName/fileType.

⏰ Set Reminder

User trigger: Click on message → "Set Reminder Alert"

Modal: ReminderModal.jsx

File: ChatApp.jsx → setReminder (function)

Fields: reminderDateTime, selectedMessage

Flow:

  1. Open ReminderModal with selectedMessage.
  2. User picks a reminderDateTime.
  3. On save, write to reminders collection with userId, messageId, remindAt, messageText.
  4. Set hasReminder: true on the message document.
  5. Close modal.

✅ Convert Message to Task

User trigger: Click on message → "Convert to Official Task"

Modal: TaskConvertModal.jsx

File: ChatApp.jsx → convertToTask

Fields: taskAssignees, taskDeadline, selectedMessage

Flow:

  1. Open TaskConvertModal with selectedMessage.
  2. User selects assignees (emails) and a deadline.
  3. On confirm, call convertToTask which updates the message document with isTask: true and taskData (deadline, assignees, status: "Pending", trail).
  4. Log audit TASK_CREATE.
  5. Play task sound.

📅 Schedule Message

User trigger: Click clock icon in input bar.

Modal: ScheduleSendModal.jsx

File: ChatApp.jsx → handleScheduleMessage

Fields: scheduleDateTime, pendingScheduledText (or inputText), activeGroup

Flow:

  1. Open modal with pendingScheduledText (the current input).
  2. User selects a future date/time.
  3. On confirm, write to scheduled_messages with text, senderEmail, groupId, scheduledFor, status:"pending".
  4. A 30‑second heartbeat in ChatApp checks for due scheduled messages and sends them (calls addDoc to messages, then updates status to "sent").

👤 Edit Profile

User trigger: Gear icon in sidebar header.

Modal: ProfileSettingsModal.jsx

File: ChatApp.jsx → handleProfileSubmit

Fields: profileForm.name, profileForm.fontSize/fontFamily, toolPreferences, optional avatar file.

Flow:

  1. Open modal pre‑filled with currentUserData.
  2. User can change display name, toggle preferences, upload avatar.
  3. Save: optional file upload to Storage, then update user doc with updateDoc.

🏢 Create / Edit Department

User trigger: "+" button in sidebar (admins) or "Create Department" on welcome screen.

Modal: GroupFormModal.jsx

File: ChatApp.jsx → handleGroupSubmit

Fields: groupForm.name, groupForm.members, groupForm.profilePicUrl (optional).

Flow:

  1. If editing, pre‑fill form with existing group data; otherwise empty.
  2. User selects members and uploads avatar (optional).
  3. On submit, create or update document in groups collection.
  4. Group includes admins (creator is first admin).

👥 View / Edit Group Members

User trigger: Click group header in chat.

Modal: GroupSettingsModal.jsx

File: ChatApp.jsx → handleUpdateGroupMembers

Fields: groupForm.members (populated from activeGroup).

Flow:

  1. Open modal showing all users; admins can toggle members.
  2. "Save Matrix" updates group doc with new member list.

📋 Task Trail (view / manage task)

User trigger: Click on a task message or from task sidebar.

Modal: TaskTrailModal.jsx

File: ChatApp.jsx (multiple handlers)

Fields: selectedMessage, taskData (deadline, assignees, status, trail).

Flows:

  • Edit objective: handleSaveTaskTitle → updates message text.
  • Transfer (delegate): handleDelegateTask → updates assignees, adds trail entry.
  • Add comment: handleAddComment → pushes trail entry with comment.
  • Attach file to task: handleTrailFileUpload → uploads file, adds trail entry with fileUrl.
  • Mark completed: handleCompleteTask → status = "Completed", trail entry.
  • Archive: handleArchiveTask → isArchived = true.

🛡️ Admin – Edit User Roles

User trigger: Pencil icon next to user in Admin panel.

Modal: AdminEditUserModal.jsx

File: ChatApp.jsx → handleEditUserSubmit

Fields: adminForm (uid, name, isAdmin, canCreateGroups).

Flow: Open modal pre‑filled, user toggles checkboxes, save updates user doc.

📊 Task Analytics

User trigger: "Analytics" button in Admin header.

Modal: TaskAnalyticsModal.jsx

Data: analyticsData computed from messages in ChatApp.

Flow: Read‑only display; no modifications.

👍 React to Message

User trigger: Hover over message bubble and click an emoji, or click on existing reaction.

File: ChatApp.jsx → handleReaction

Fields: message.reactions map.

Flow: Toggle user's email in emoji's array, update doc, notify original sender if not self.

📌 Pin / Bookmark Message

User trigger: Hover action buttons (thumbtack / bookmark).

File: ChatApp.jsx → handleTogglePin, handleToggleBookmark

Flow: Toggle isPinned or add/remove from bookmarkedBy array.

Security Rules (Production)
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{uid} {
      allow read, write: if request.auth.uid == uid || get(/databases/$(database)/documents/users/$(request.auth.uid)).data.isAdmin == true;
    }
    match /messages/{msgId} {
      allow read: if resource.data.allowedUsers.size() == 0 || request.auth.token.email in resource.data.allowedUsers;
      allow create: if request.auth.uid == request.resource.data.senderUid;
      allow update, delete: if request.auth.uid == resource.data.senderUid || get(...).data.isAdmin == true;
    }
    match /groups/{groupId} {
      allow read: if request.auth.token.email in resource.data.members;
      allow create: if request.auth.uid != null;
      allow update, delete: if request.auth.token.email in resource.data.admins || get(...).data.isAdmin == true;
    }
    match /{document=**} { allow read, write: if false; }
  }
}

For full security, also protect reminders, notifications, audit_logs with user‑specific rules.

Future Roadmap & Enhancements
📱 Android App

Wrap the existing PWA with Capacitor to generate a native Android APK/AAB. Same codebase, no rewrite. Add push notifications via FCM, publish on Google Play.

💰 Subscription Model

Integrate Stripe using the Firebase “Run Subscription Payments” extension. Gate premium features (analytics, templates, unlimited groups, larger file uploads) behind plans (Free, Pro, Team). Show upgrade prompts in the UI.

🏢 SaaS / Multi‑Tenancy

Introduce organizations collection. Each org has its own users, groups, tasks, and branding. Users sign up with their company email and are auto‑assigned to the correct tenant. Admin can white‑label the interface (logo, primary color).

📈 Scalability
  • Message pagination – infinite scroll with Firestore cursors (limit 50 per page).
  • Presence via Realtime Database – reduce Firestore writes for typing/heartbeat.
  • Cloud Functions – move sensitive operations (admin promotion, task reminders, scheduled sends) to secure serverless functions.
  • Composite indexes – create for common queries (tasks by assignee + status).
🌙 UI & Theme Improvements
  • Dark mode – toggleable with Tailwind’s `dark:` variant, persisted in user preferences.
  • Custom themes per user/org – primary color, font size presets.
  • Rich formatting toolbar – bold, italic, lists, code blocks in messages.
  • Improved file preview – image lightbox, PDF viewer, Office document thumbnails.
⚡ Speed & Performance
  • Code splitting & lazy loading – load admin/modals only when needed.
  • Image optimization – resizing uploaded images before upload, using WebP format.
  • CDN caching – for static assets (icons, sounds).
  • Bundle analyzer – keep bundle size under 200KB gzipped.
🧩 New Features (inspired by top task apps)
  • Task priority – High / Medium / Low with visual flags.
  • Labels / Tags – categorize tasks (e.g., “urgent”, “design”, “bug”).
  • Recurring tasks – enhanced from templates: daily, weekly, custom CRON.
  • Google Calendar integration – push task deadlines to user’s Google Calendar via API.
  • Email from chat – “Send as email” button that opens a compose window or sends via SendGrid/Postmark.
  • Email digests – daily/weekly summary of pending tasks and unread messages.
  • Voice messages – record & send short audio clips inline.
  • Message threads – threaded conversations under a main message.
  • External sharing – share a read‑only link to a task or conversation with clients.
  • Two‑factor authentication (2FA) – enhance security for admin accounts.
